# nx

NixOS and Home Manager dotfiles for `zen`, with shared system/home modules and host-specific disk layout files.

## Architecture

```text
~/nx
├── flake.nix
├── hosts/
│   └── zen/
│       ├── default.nix        # NixOS entrypoint
│       ├── hardware.nix       # generated by nixos-generate-config
│       ├── disko.nix          # declarative disk layout
│       └── home.nix           # Home Manager entrypoint
├── modules/
│   ├── nixos/                # shared NixOS modules
│   └── home/                 # shared Home Manager modules
└── wallpapers/
```

## Common Commands

```bash
nix fmt .
nix flake check

home-manager build --flake ~/nx#wyattgill@zen
home-manager switch --flake ~/nx#wyattgill@zen

sudo nixos-rebuild build --flake ~/nx#zen
sudo nixos-rebuild switch --flake ~/nx#zen
```

## Quick Start For A New Host

1. Copy `hosts/zen/` to `hosts/<new-host>/`.
2. Replace `hardware.nix` and `disko.nix` with the new machine's values.
3. Add a new host definition in `flake.nix` with its `hostName`, `username`, `email`, and `homeDirectory`.
4. Point a `nixosConfigurations.<new-host>` entry at `./hosts/<new-host>/default.nix`.
5. Add a matching `homeConfigurations."user@host"` entry for `./hosts/<new-host>/home.nix`.
6. Run `nix fmt .`, `nix flake check`, `home-manager build --flake`, and `sudo nixos-rebuild build --flake` before switching.

## Notes

- `modules/home/desktop/common.nix` is the shared source of truth for desktop session packages.
- `modules/nixos/nix.nix` uses `nix-index-database`, disables channels, and keeps Nix fully flake-first.
- `modules/home/terminal/ai.nix` sources AI tools from `github:numtide/llm-agents.nix`.
